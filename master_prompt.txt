### PROJECT SNAPSHOT (current system state)

- React + Vite project with react-three-fiber and drei
- 3D exploding box with:
  - animated lid (spring)
  - animated side panels (hinged spring panels)
  - realistic cardboard material (MeshStandardMaterial)
  - PaperInset system (fit / tileSquare / tileBase)
  - 2 mm inset for paper
  - texture caching
  - camera controls via OrbitControls

### LIGHTING (critical feature)
- Using FixedLights.jsx
- Lights are aligned to camera orientation every frame, so the lighting stays “from the front” while object rotates
- Includes:
  - front key light
  - right fill light
  - left fill light
  - top soft light
- All lights have castShadow = false

### TONE MAPPING
- LinearToneMapping
- exposure = 1.25
- sRGB color space

### WHAT I WANT TO DO NEXT
FIX TIMING AND ORDER of opening the box. The lid must finish its animation and be at its final position before the walls open. Currently walls open at the same time the lid is starting to move. 
------------------------------------------------------------
Official sequence (from DOM events in Scene.jsx):

"open-box":
   1) lidSpring.open animate 0 → 1
   2) wallsSpring.open animate 0 → 1 (after lid completes)

"close-box":
   1) wallsSpring.open animate 1 → 0
   2) lidSpring.open animate 1 → 0

"reset-box":
   instantly sets both springs to 0

This ensures:
- lid opens first, then walls unfold
- walls fold first, then lid closes
- no jarring intersections
- smooth, physically readable motion
