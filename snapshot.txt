4. NEW ANIMATION SYSTEM (2026)
============================================================
Two value springs:
- lidSpring.open  (0 → 1)
- wallsSpring.open (0 → 1)

Controlled by BoxAnimator.js:
- open()
- close()
- reset()

Key change:
React‑spring does NOT delay visual movement when using duration,
so manual timing delays were introduced for **real** sequencing:

OPEN:
  1) lidSpring.open = 1
  2) wait(ANIM.lid * 600)
  3) wallsSpring.open = 1

CLOSE:
  1) wallsSpring.open = 0
  2) wait(ANIM.walls * 500)
  3) lidSpring.open = 0

RESET:
  - Both springs snap instantly to zero

BoxAnimator.js includes:
- animation lock (prevents overlap)
- reusable wait(ms) helper
- isolated animation logic (Scene is clean now)

============================================================
5. SPRING USAGE
============================================================
Springs are physics-based:
- tension 110–120
- friction 16–18
- Manual delay for visual ordering

This solves:
- walls opening too early
- lid closing too early
- race conditions
- inconsistent animation overlap
